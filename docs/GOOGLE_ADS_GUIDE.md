## 1. Stratégie de Campagne — Structure

**Type** : Campagne Search uniquement (pas Display, pas Performance Max au démarrage)
**Géo** : France uniquement
**Langue** : Français
**Budget total départ** : 7€/jour (réparti sur 2 AdGroups actifs)
**Enchères** : CPC Manuel — NE PAS activer Smart Bidding avant 30 conversions `purchase` enregistrées

### Structure recommandée : 2 AdGroups actifs + 1 en veille

| AdGroup | Intention | Budget/jour | Priorité |
|---------|-----------|-------------|----------|
| **AG1 — Problème ATS** | Candidat qui sait que son CV est bloqué par un algorithme | 4€ | ★★★ Priorité 1 |
| **AG2 — Optimisation CV** | Candidat qui cherche à améliorer son CV sans connaître ATS | 3€ | ★★ Priorité 2 |
| **AG3 — Remarketing Display** | Visiteurs ayant uploadé un CV sans acheter | 0€ (activer semaine 3) | En veille |

## 5. Plan de pilotage — Semaines 1 à 4

**Objectif** : Laisser Google collecter des données. Aucune modification d'enchères.

Checklist à faire UNE SEULE FOIS le jour du lancement :
- [ ] URL finale : `https://careermatch.fr/lp/cv-ats?utm_source=google&utm_medium=cpc&utm_campaign={campaign}&utm_content={adgroup}&gclid={gclid}`
- [ ] Vérifier que la balise de conversion se déclenche dans Tag Assistant (Chrome extension)
- [ ] Vérifier que `begin_checkout` remonte dans GTM Preview
- [ ] Mettre en ligne AG1 uniquement — AG2 en veille
- [ ] Activer AG2 à J+3 si AG1 consomme bien le budget

Indicateurs à surveiller (lecture seule) :
- CTR > 3% = titres pertinents
- CTR < 1% = titres à revoir
- Taux de rebond Landing Page dans GA4 (ou Clarity) < 70%

### Semaine 2 — Première lecture

**Actions autorisées** :
- [ ] Couper les mots-clés avec > 20 clics et 0 conversion `sign_up`
- [ ] Identifier les search terms réels (Rapport → Termes de recherche) et ajouter les termes hors-cible en négatifs
- [ ] Augmenter CPC de +20% sur les mots-clés ayant généré des `sign_up`

**Indicateurs clés** :
| Métrique | Seuil "Couper" | Seuil "Scaler" |
|----------|----------------|----------------|
| CTR | < 1.5% | > 5% |
| CPC réel | > 2.50€ | < 0.80€ |
| Conversions sign_up | 0 après 30 clics | ≥ 1 pour 15 clics |

### Semaine 4 — Décision Scale ou Pivot

**Scénario A — ROAS > 2x (revenus > 2x budget pub)** :
- Augmenter budget de +50%
- Activer Smart Bidding : Target CPA = 8€ (si panier moyen ~10€)
- Créer AG4 : mots-clés longue traîne spécifiques par métier ("cv data analyst ats", "cv développeur ats")

**Scénario B — ROAS < 1x** :
- Couper AG2, concentrer budget sur AG1
- Revoir les titres (A/B test radical : changer 5 titres sur 15)
- Vérifier dans Clarity si les visiteurs utilisent vraiment le QuickScan

**Scénario C — Beaucoup de sign_up, peu de purchase** :
- Problème de conversion post-inscription, pas de pub
- Vérifier l'email de bienvenue (reçu ?)
- Vérifier que le modal d'upsell s'affiche bien quand crédits = 0

## 6. Rappel Tracking — 3 Points Critiques à Vérifier Après Lancement

### Point 1 — Conversion ID et Label
Aller dans **Google Ads → Outils → Conversions → "Achat Career Match"**

Vérifier :
- Conversion ID : `18330710463`
- Label : `hBNlCO0podlcEL_j4aRE`
- Type de valeur : **Variable** (pas fixe)
- Fenêtre d'attribution : 30 jours clic / 1 jour vue

Le statut affichera "Mauvaise configuration" jusqu'à la **première vraie conversion**. C'est normal. Il passera à "Opérationnel" automatiquement après le premier achat.

### Point 2 — Vérification Consent Mode v2 (obligatoire EEE)

Le site implémente le Consent Mode v2 natif. Avant de lancer, vérifier :

1. Ouvrir `careermatch.fr` en navigation privée (localStorage vide)
2. Dans Tag Assistant, observer que `ad_storage = denied` par défaut
3. Cliquer "Tout accepter" sur la bannière cookies
4. Observer que GTM reçoit un hit `consent` avec `ad_storage = granted`

Si `ad_storage` reste `denied` après consentement → vérifier que `consent.ts` appelle bien `gtag('consent', 'update', ...)`.

**Dans GTM** : Admin → Consent Overview → confirmer que les balises Google Ads ont le statut "Consent checked".

---

### Point 4 — Test du Tag Achat en conditions réelles
Procédure de test (à faire avant de dépenser 1€ de pub) :

1. Installer l'extension Chrome **Google Tag Assistant**
2. Ouvrir `careermatch.fr/lp/cv-ats?utm_source=google&utm_medium=cpc&gclid=TEST123`
3. S'inscrire avec un compte test
4. Aller sur la page Tarifs (`/app` → step 7)
5. Cliquer "Obtenir 20 Crédits"
6. Dans Tag Assistant, vérifier que la balise **"Google Ads — Conversion Achat"** s'est déclenchée avec `value: 4.99`

Si elle ne se déclenche pas → vérifier dans GTM que le déclencheur `Event - purchase` écoute bien l'événement `purchase` (pas `Purchase` avec majuscule).

### Point 5 — Délai d'activation Smart Bidding
**NE PAS activer Target CPA ou Target ROAS avant :**
- 30 conversions `purchase` enregistrées dans les 30 derniers jours
- Minimum 2 semaines de données

Avant ce seuil, Smart Bidding est aveugle et surpaye chaque clic. Rester en CPC manuel jusqu'à ce volume.
